Case Overview

WALKOFF contains a flexible, customizable logging system. It's better than your average, fatal/error/warning/info/trace logging system -- it's a database-driven event subscription system which allows you to visualize the workflow execution at any level of detail you deem useful. You start by creating and naming one or more "cases" which are collections of execution events you wish to log together, we call these the case's event subscriptions. As a workflow or collection of workflows is executed, the events to which the case is subscribed to are logged. Each level of execution contains multiple events which can be subscribed to. Currently the available event subscriptions are as follows:

Controller
  • Scheduler Started
  • Scheduler Paused
  • Scheduler Resumed
  • Scheduler Shutdown
  • Job Added
  • Job Removed
  • Job Executed
  • Job Error
Workflow
  • Workflow Execution Pending
  • Workflow Execution Start
  • App Instance Created
  • Workflow Arguments Validated
  • Workflow Arguments Invalid
  • Workflow Paused
  • Workflow Resumed
  • Workflow Shutdown
  • Workflow Aborted
Step
  • Action Execution Success
    • ,
  • Action Execution Error
  • Action Started
  • Arguments Invalid
  • Trigger Action Awaiting Data
  • Trigger Action Taken
  • Trigger Action Not Taken
Branch
  • Branch Taken
  • Branch Not Taken
Conditional Expression
  • Conditional Expression True
  • Conditional Expression False
  • Conditional Expression Error
Condition
  • Condition Success
  • Condition Error
Transform
  • Transform Success
  • Transform Error

Each case can be subscribed to multiple workflows. Each execution element can have multiple execution elements it manages, and for each execution element instance, a case which can have an individualized set of events it is subscribed to. So if you want one step in one workflow to be log the events Action Execution Success and Conditionals Executed, but a different step in that same workflow to only log its Arguments Validated event, you can specify these.

Each event is logged with the following fields

Unique ID
Timestamp
Type
System, Workflow, Action, Branch, Conditional Expression, Condition, Transform
Originator
A UUID of the execution elements which caused this event
Message
The specific event that happened
Note
Field which can be modified to add annotations
