Tag Description 
workflows
Container for Workflows 
start

The name of the start step 

steps

Container for step objects 

uid

UUID of an element 

app

App Name 

name

Name of the playbook or workflow or step 

action

Action name to be taken

 
device

Device Name (configuration profile) to be used 

inputs

Container for arguments

Tags contained within are specific to each app. 

next

Container for conditional

step=Step to jump to next if conditional is true 

flags

Conditional statement object

action=function to verify output against 

args

Container for conditional arguments

Elements contained within are specific to function called

 
filters

Container for filter objects

 
status

Status to use for the next step. Steps can return multiple status codes (default is "Success") 

position

Position if teh step node on the workflow editor
