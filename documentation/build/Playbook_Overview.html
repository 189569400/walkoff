Tag Description 
workflows
Container for Workflows 
start

The UID of the start step 

actions

Container for step objects 

id

UUID of an element 

app_name

App Name 

name

Name of the playbook, workflow, or action 

action_name

Name of the action, condition, or transform to be taken

 
device_id

Device (configuration profile) ID to use 

trigger

Conditional Expression to be evaluated against data sent to a URL before a step takes an action 

condition

Conditional Expression to be evaluated against data sent to a URL before a step takes an action 

arguments

Container for arguments to actions

 
value

Static value of an argument

 
reference

UUID of a previously-executed action to use as an argument

 
selection

The path to a element of a list or JSON object to use an argument

 
branches

Container for conditional objects

 
conditions

Container for Condition objects

 
child_expressions

Container for Condition Expression objects

 
transforms

Container for transform objects

 
is_negated

Should the result from the Condition or Conditional Expression be negated? Meaning True becomes False and vice versa

 
status

Status to use for the branch. Steps can return multiple status codes (default is "Success") 

priority

Priority of the branch. A branch with a priority of 1 is evaluated first. 

source_id

The UUID of the source action of the branch

 
destination_id

The UUID of the destination action of the branch

 
position

Position if the action node on the workflow editor
