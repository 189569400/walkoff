The Scheduler

  • Open then Scheduler page. This page will display the scheduled tasks and allow you to run or stop the scheduler itself or the individual tasks.
    You can add a new task by selecting the "+Add To Schedule" button on the top left. On this window, you can configure a new task. You can add multiple workflows to a scheduled task.
    Give the task a name and a helpful description of what it does.
    You can select either "date", "interval", or "cron" type of schedule trigger. The simplest of the three is the date trigger.
    Trigger Type Effect
    date Executes the workflow(s) once at a given date and time.
    interval Executes a workflow periodically (ex. every 2 hours 30 seconds). A starting date and time are optional.
    cron The most complex of the three and similar to a Linux cron-job. You can use this to schedule tasks such as "every third wednesday in even-numbered months at 10:30am." The parameters are complex. In the backend, we use Python APScheduler. The parameters allowed are the same as for the CronTrigger object of this library, and you can see their documentation here.
    In this example, we use the interval trigger. You can select a start date.
    And a time
    Once you add this task, you'll be able to see it on the main page.
    By default, a task starts running as soon as it is created. However, the scheduler needs to be started manually. You can start the scheduler By clicking the play button above the table.
    Stopping the scheduler will stop all tasks executing. You can also stop a task by clicking the stop button on the right hand side of the row.
