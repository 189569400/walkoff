Branching Workflow Creation Tutorial

In this tutorial, we will create a Workflow with branching paths that utilizes flags and filters.

  • Create a new Workflow

    First, begin by creating a new Workflow. Enter the Playbook name and workflow name, then click Create New Workflow.


  • Add actions to the workspace

    In this workflow, we will use the "Secure PRNG" action from the included Utilities app, and three "Repeat Back to Me" actions. Ensure that the "Secure PRNG" action is set as the start node, and connect the actions as shown.


  • Configure actions

    The "Secure PRNG" action will output a random floating point number between 0.0 and 1.0 using system entropy (it's really only as secure as your system's implementation). We want to know whether the number it outputs is less than, or greater than/equal to 0.5.

    Select the "Secure PRNG" action and find the Next Nodes section in the right pane. In one of the Next Node entries, set the return code to "Success", and the flag as "count" with inputs of "l" and "0.5". In the other, set the return code and flag as appropriate.

    Select the first "Repeat Back to Me" whose next node entry you configured above, and enter "less than 0.5" as the input. In the other "Repeat Back to Me" action, enter "greater than or equal to 0.5."

    Finally, in the third "Repeat Back to Me" action, set the input as @Name, where Name is replaced by the name of the "Secure PRNG" action.

    Then, save and execute the workflow.


  • Examine results

    If you configured everything as shown, you should obtain similar results to what is shown. The correct "Repeat Back to Me" action should have executed according to the output generated by the "Secure PRNG" action, and the last "Repeat Back to Me" action should have repeated that number. Run the Workflow a few more times to verify that it was set up correctly.
