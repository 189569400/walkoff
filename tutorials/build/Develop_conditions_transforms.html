Develop Conditions and Transforms

Developing conditions and transforms is similar to developing any other app action, with only a couple key differences:

  1. Conditions and transforms must be global
  2. Conditions and transforms must specify an additional parameter for the data supplied by the previous action
  3. Conditions and transforms use their own decorators
  4. Conditions must return True/False and do not specify return codes
  5. Transforms cannot return any return codes other than "Success"

In this tutorial we'll develop some simple conditions and transforms.

Develop a condition

Inside our HelloWorld app, we create a module named conditions.py. Inside this module, we will define a condition called list_contains which will take a list of strings from the previously executed action or transform and return true if a given string is found in that list.


from apps import condition

@condition
def list_contains(list_in, member):
    return member in list_in

And we add this condition to our api.yaml file:


walkoff: '0.1'
info:
  title: HelloWorld
  version: 1.0.0
actions:
  # ... actions go here ...
conditions:
  list contains:
    run: conditions.list_contains
    description: Returns true if the list contains the given member
    data_in: list_in
    parameters:
      - name: list_in
        description: The list to search through
        type: array
        items:
            type: string
        required: true
      - name: member
        description: The string to search for
        type: string
        required: true

The data_in field of the metadata file specifies the parameter to which the data from the previously executed transform or action will be sent

Develop a transform

Inside our HelloWorld app, we create a module named transforms.py. Inside this module, we will define a transform called decimal_to_hex which will take a decimal integer from the previously executed transform or action and return its hexidecimal representation using Python's builtin hex() function.


from apps import transform

@transform
def decimal_to_hex(decimal_in):
    return hex(decimal_in)

And we add this transform to our api.yaml file:


walkoff: '0.1'
info:
  title: HelloWorld
  version: 1.0.0
actions:
  # ... actions go here ...
transforms:
  decimal to hex:
    run: transforms.decimal_to_hex
    description: Returns hexidecimal value of the given integer
    data_in: decimal_in
    parameters:
      - name: decimal_in
        description: The number to convert to hexidecimal
        type: integer
    returns:
      Success:
        schema:
            type: string
            description: The hexidecimal representation of the number passed in

Just like in the condition, the data_in field of the metadata file specifies the parameter to which the data from the previously executed transform or action will be sent
