Workflow Editor Navigation Tutorial

In this tutorial, we will explore the different components of the Workflow Editor interface.

  • Toolbar

    From left to right, the buttons in the toolbar are:

    Name Descrption
    New Workflow Creates a new Workflow and optionally a new Playbook.
    Open Workflow Opens an existing Workflow
    Save Workflow Saves current Workflow to Playbook.
    Delete Selected Nodes Deletes the selected Action or Edge.
    Cut Cuts the selected Action or Edge.
    Copy Copies the selected Action or Edge.
    Paste Pastes the previously copied or cut Action or Edge.
    Execute Workflow Schedules the Workflow to run immediately.
    Clear Execution Results Highlighting Clears the green highlighting of executed actions.

    The Delete, Cut, and Copy tools can act on multiple Actions and Edges at a time if they are selected by holding the Shift key and clicking on multiple Actions or Edges.


  • Highlighting

    Selected Actions or Edges will be highlighted in blue. Actions that have been executed during a run will be highlighted in green.


  • Apps and Actions

    Apps and Actions that you have installed to your Walkoff instance will appear in the left pane. Double-click the name of an App or click the drop-down arrow to expose the Actions that App contains. To take an Action from the pane, double click on the name of the action or click and drag the action into the workspace.


  • Navigating the Workspace

    The workspace can be navigated using the buttons in the top left of the pane. From top to bottom, these buttons can be used to pan, fullscreen, or zoom. You can also click and drag on an empty area to pan, and use the scroll wheel to zoom in and out.


  • Connecting actions together

    When hovering over the top edge of an action, a red dot will appear. To create an edge from one action to the next, click and drag from the red dot to the top of the next action in the sequence; an arrow should appear, linking the actions together.


  • Editing Actions

    When an Action is selected, its properties will appear in the right pane.

    The Node Parameters will include the Name of the action (which is separate from its UID), the App it came from, the Action it will perform, and the Device it is associated with, if any. You can also designate the start node of the Workflow in this section.

    Actions may have inputs, whose properties are defined by the App's API schema. To use the output from an Action as an input to another Action, use @Name, where Name is the name of the outputting action.

    Next, if the Action is not terminating (i.e. it has edges pointing away from it), the Next Nodes section will allow you to set the conditions under which Workflow execution will continue to the next node(s). This can be accomplished with return codes, filters and flags.

    Actions emit return codes when they finish executing. By default, an Action can return Success, UnhandledException, or InvalidInput. Additional return codes can be defined in an App's schema.

    Filters allow you to manipulate the output from the current Action (without changing it), for example selecting a specific field in a JSON output, or performing mathematical transformations.

    Finally, Flags allow you to check an action's output after any filters, for example, against conditional operators (greater than, less than, etc.) or against a regular expression.


  • Evaluating results

    When a Workflow in the editor is executed, actions are highlighted in green as they execute in realtime. When the workflow is complete, its results will be displayed in the table below the editor. The table will contain the name of the node, the time at which it finished executing, the return code, any inputs to the action, and the results of the action.
