Workflow Editor Navigation Tutorial

In this tutorial, we will explore the different components of the Workflow Editor interface.

  • Toolbar

    From left to right, the buttons in the toolbar are:

    Name Descrption
    New Workflow Creates a new Workflow under a new or existing Playbook.
    Open Workflow Opens an existing Workflow
    Save Workflow Saves current Workflow to Playbook.
    Delete Selected Nodes Deletes the selected Action or Edge.
    Undo Reverts the most recent change in the editor.
    Redo Reapplies the most recent undone action.
    Cut Cuts the selected Action.
    Copy Copies the selected Action.
    Paste Pastes the previously copied or cut Action.
    Execute Workflow Schedules the Workflow to run immediately.
    Open Execution Results Opens the table of results for the current workflow.
    Clear Execution Results Highlighting Clears the green/red highlighting of executed actions.

  • Highlighting

    Selected Actions or Edges will be highlighted in blue. Actions that have been executed successfully during a run will be highlighted in green, while ones that have encountered errors will be highlighted in red.


  • Apps and Actions

    Apps and Actions that you have installed in your Walkoff instance will appear in the left pane. Click the name of an App to reveal the Actions that App provides. To use an Action from the pane, double click on the name of the Action or click and drag the Action into the workspace.


  • Navigating the Workspace

    The workspace can be navigated using the buttons in the top left of the pane. From top to bottom, these buttons can be used to pan, zoom to fit, or zoom in and out. You can also click and drag on an empty area to pan, and use the scroll wheel to zoom in and out.


  • Connecting actions together

    When hovering over the top edge of an action, a red dot will appear. To create an edge from one action to the next, click and drag from the red dot to the top of the next action in the sequence; an arrow will appear, linking the actions together.

    An action can point to more than one action; these are called branches.


  • Branch configuration

    When an action is connected to more than one subsequent action, the path the workflow takes will depend on the branch parameters that have been set. At this time, only one branch at a time will be taken; concurrent branches are not supported.

    Branches can be controlled by the return type of the preceeding action, a priority number, as well as custom conditions that can be defined within Apps.


  • Editing Actions

    When an Action is selected, its properties will appear in the right pane.

    The Action Parameters will include the App it came from, the Action it will perform, the Name of the Action (separate from its unique ID), and the Device it is associated with, if any. You can also set the starting Action of the Workflow in this pane.

    Actions may have arguments, whose properties are defined by the App's API schema. By default, an argument is a static value that you specify. To use the output from an Action as an argument to another Action, select the action's name in the dropdown menu. If an Action returns an object, you can use dot (object) and bracket (array) notation to drill down.

    Actions can wait on Trigger Conditions to fire before executing. To use one, select the App and Action to be used as a condition, then click "Add Condition" and fill out the appropriate arguments.

    Actions can also transform their outputs prior to processing conditions. To use a transform, select the App and Action to be used, click "Add Transform", and fill out the appropriate arguments.


  • Evaluating results

    When a Workflow in the editor is executed, actions are highlighted in green (if successful) or red (if failed) as they execute in realtime. When the workflow is complete, its results will be displayed in a table accessed by the "Open Execution Results" button. The table will contain the name of the Action UID, Action Name, the time at which it finished executing, the return code, any arguments to the action, and the results of the action.
