Managing app dependencies

WALKOFF allows you to reduce the number of dependencies needed by allowing you to selectively install apps.

Specifying app dependencies

WALKOFF already requires several dependencies to run. There are listed in the `requirements.txt` file in the root directory of the repository. However, you may develop an app which needs additional dependencies. For example, if you were developing an app which incorporated machine learning, your app my depend on the popular scikit-learn Python package. However, this package has several other dependencies, and if a user didn't need your app, we can spare them needing to install them by including them in them in your app's own requirements.txt file. For our example, our requirements.txt would look like this:


scikit-learn

Installing app dependencies

To install an app dependency, you can use the installDependencies.py script included at the top level of the repository.


python installDependencies.py --apps App1

You can see more options using 


python installDependencies.py --help
