Using the WALKOFF API (Execution)

One of the primary uses of the WALKOFF API will be to execute workflows and send data to actions in workflows that are awaiting data. In this part of the WALKOFF API tutorial, having authenticated with the WALKOFF instance and verified the existence of our workflow, we will execute it and check the results of the workflow execution.

At the end of this tutorial series, we will have developed a script that:

  1. Takes in address and username arguments for a WALKOFF instance.
  2. Attempts to authenticate with the WALKOFF instance at that address with the specified username, prompting for a password.
  3. Refreshs the access token.
  4. Lists the available workflows.
  5. If the workflow "Tutorial" exists, executes it.
  6. Checks the results of the workflow.
  7. Logs out.

This part will detail steps 5 and 6.

1. Execute a workflow.

To execute a workflow, we'll POST to the /api/workflowqueue endpoint.


def execute_workflow(self, workflow_id):
    execution_uid = None
    if self.is_connected:
        print("Executing workflow..."),
        ep = "/api/workflowqueue"

        r = requests.post(self.url + ep, headers=self.auth_header, json={"workflow_id": workflow_id},
                          timeout=2)

        if r.status_code == 202:
            execution_uid = r.json()["id"]
            print("Workflow: {} was executed with execution ID: {}".format(workflow_id, execution_uid))

        elif r.status_code == 404:
            print("Workflow does not exist.")

        else:
            print("Unknown response: {}: {}".format(r.status_code, r.json()))

    return execution_uid

If the workflow successfully begins executing, it will return the execution_uid for that run, which can be used to further interact with it.

2. Check workflow results

To retrieve the results of a workflow, query the /api/workflowqueue/{execution_uid} endpoint.


def check_result(self, execution_uid):
    success = False
    if self.is_connected:
        print("Checking results..."),

        ep = "/api/workflowqueue/{}".format(execution_uid)

        r = requests.get(self.url + ep, headers=self.auth_header, timeout=2)

        if r.status_code == 200:
            success = True
            print("Results of {}: {}".format(
                execution_uid, r.json()))

        else:
            print("Unknown response: {}: {}".format(
                r.status_code, r.json()))
    return success

The /api/workflowqueue/{execution_uid} endpoint takes the execution_uid for the targeted workflow run in the URI, and returns an object with all execution details and results from that run.

The sleep call is to ensure that the server has enough time to completely execute the workflow before we check its results.
