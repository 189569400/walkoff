Branching Workflow Creation Tutorial

In this tutorial, we will create a Workflow with branching paths that utilizes flags and filters.

  • Create a new Workflow

    In the top navigation bar, click "Playbook" to return to the playbook editor. Then, in the toolbar, click the "New Workflow" button. You can specify an existing Playbook or create a new one. Enter or select the Playbook name and enter the Workflow name, then click "Create New Workflow."


  • Open the newly created Workflow

    In the toolbar, click the "Open Workflow" button. You can click Playbook names to expand or collapse them. You can also rename, duplicate, and delete workflows here. Click on the name of the Workflow that was just created.


  • Add actions to the workspace

    In this workflow, we will use the "System PRNG" action from the included Utilities app, and two "repeatBackToMe" actions. Ensure that the "System PRNG" action is set as the start node, and connect the actions as shown.


  • Configure actions

    The "System PRNG" action will output a random floating point number between 0.0 and 1.0 using system entropy (it's really only as secure as your system's implementation). We want to know whether the number it outputs is less than, or greater than/equal to 0.5.

    Select the left branch, set the return code to "Success" and priority to 1, Under Conditions, select "Utilities" and "Number Compare", then set a threshold of 0.5 and a "ge" operator for "greater than or equal". In the right branch, set the same settings, but use an "l" operator for "less than" instead.

    Select the left "repeatBackToMe" action, and enter "ge to 0.5" as the string argument. In the right "repeatBackToMe" action, enter "l than 0.5" instead.

    In both "repeatBackToMe" actions, be sure to set the device as configured in the previous tutorial.

    Then, save and execute the workflow.


  • Examine results

    If you configured everything as shown, you should obtain similar results to what is shown. The correct "repeatBackToMe" action should have executed according to the output generated by the "System PRNG" action. Run the Workflow a few more times to verify that it was set up correctly.
