Basic Workflow Creation Tutorial

In this tutorial, we will create our first, very basic workflow in order to demonstrate the general features of the workflow editor.

  • Create a new device

    Before we create a workflow, we will create a device that our actions will use.

    In the top navigation bar, click on "Devices", then "Add Device". Fill out any required fields (their exact values will not matter), and click "Add Device" in the dialog box.


  • Create a new Workflow

    In the top navigation bar, click "Playbook" to return to the playbook editor. Then, in the toolbar, click the "New Workflow" button. You can specify an existing Playbook or create a new one. Enter or select the Playbook name and enter the Workflow name, then click "Create New Workflow."


  • Open the newly created Workflow

    In the toolbar, click the "Open Workflow" button. You can click Playbook names to expand or collapse them. You can also rename, duplicate, and delete workflows here. Click on the name of the Workflow that was just created.


  • Add actions to the workspace

    Let's begin by adding a Hello World action and a Pause action from the HelloWorld app. Expand the HelloWorld app by clicking on the app name in the left pane. Then, double-click, or click and drag the desired actions into the workspace.
    Ensure that the Hello World action is set as the starting node by clicking "Set as Start Action" in the Action Parameters pane.


  • Configure actions

    Some actions will have inputs; some required, some optional. In this case, both actions require a device; set it to the example device created previously. Additionally, the Pause action requires you to input the time in seconds that it should pause for. Enter 5.

    Finally, connect the actions together by clicking and dragging from the top of the helloWorld action to the top of the pause action.


  • Save and Execute the Workflow

    Use the Save button and the Execute button in the toolbar.


  • Examine results

    Click on the "Open Execution Results" button in the toolbar.

    If you configured everything as shown, you should obtain similar results to what is shown. Note the 5 second delay created by the Pause action under the Timestamp heading, and the results created by each step.
