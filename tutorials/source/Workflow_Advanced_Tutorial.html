Advanced Workflow Creation Tutorial

In this tutorial, we will create a more extensive Workflow with multiple branching paths that is a little more analogous to something that might be used in production.

  • Create a new Workflow

    First, begin by creating a new Workflow. Enter the Playbook name and workflow name, then click Create New Workflow.


  • Add actions to the workspace

    In this workflow, we will use actions from the included Utilities app and the FileUtilities app which can be found in the Apps repository. The outline of the workflow is as follows:

    1. Take information from an action. In this example, it will be a random number from the System PRNG action, but it could be retrieving logs, messages, or other information.
    2. If the number is under 0.5, we will proceed with storing it to a file, otherwise it will only be logged in console.
    3. Normalize the data by rounding it to 3 decimal points.
    4. Check if the file "test.txt" exists
    5. Then if it does, output the metadata of the file and make it writable before appending the information to it. If it doesn't exist, we will first create the file before writing to it.
    6. Then, the file will be marked as read-only and output the metadata again.

    Drag the appropriate actions to the workspace, connect them as shown, and ensure that the System PRNG action is the starting node.


  • Configure actions

    1. System PRNG: Set the Next Node flag for "repeatBackToMe" as "count" with inputs "ge" and "0.5", and the flag for "Round to N" as "count" with inputs "l" and "0.5".
    2. repeatBackToMe: Set the string as "greater than or equal to 0.5".
    3. Round to N: Set the number to round as "@Name" (where Name is the Name of the "System PRNG" action), and the places to "3". The "@Name" syntax is used to pass outputs from other actions as inputs.
    4. Exists in Directory: Set the Next Node flag for "create file" as "NotExists" and the flag for "stats" as "Exists."
    5. Stats: Set the file name as "test.txt"
    6. Create File: Set the file name as "test.txt", the contents as "new file", and overwrite to "false".
    7. Make Writable: Set the file name as "test.txt"
    8. Append File: Set the file name as "test.txt", the contents as "@Name" (where Name is the Name of the "Round to N" action), and newline as "True".
    9. Make Read Only: Set the file name as "test.txt"
    10. Stats: Set the file name as "test.txt"

    Once you have all nodes configured as above, save and execute the workflow.


  • Examine results

    If you configured everything as shown, you should obtain similar results to what is shown. Take a look at the generated "text.txt" if it gets generated. Run the Workflow a few more times to verify that it was set up correctly.
