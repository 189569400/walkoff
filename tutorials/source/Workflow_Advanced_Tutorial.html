Advanced Workflow Creation Tutorial

In this tutorial, we will create a more extensive Workflow with multiple branching paths that is more analogous to something that might be used in production.

  • Create a new Workflow

    First, begin by creating a new Workflow. Enter the Playbook name and workflow name, then click Create New Workflow.


  • Add actions to the workspace

    In this workflow, we will take information from an action (in this example, it will be a random number from the Secure PRNG action, but it could be retrieving logs, messages, or other information), normalizing the data (rounding it to 5 decimal points), then appending it to a file. We will first check if the file exists, then if it does, output the metadata of the file and make it writable before appending the information to it. If it doesn't exist, we will first create the file before writing to it. Then, the file will be marked as read-only and output the metadata again. If, when creating or writing to the file fails for whatever reason, we will be notified.
    Drag the appropriate actions to the workspace, connect them as shown, and ensure that the Secure PRNG action is the starting node.


  • Configure actions

    The Truncate action requires an input determining the number of decimal points to round to. Enter 5.
    The Exists in Directory action requires a file name to check for.


  • Save and Execute the Workflow


  • Examine results

    If you configured everything as shown, you should obtain similar results to what is shown. The correct "Repeat Back to Me" action should have executed according to the output generated by the "Secure PRNG" action, and the last "Repeat Back to Me" action should have repeated that number. Run the Workflow a few more times to verify that it was set up correctly.
