Workflow Overview


<?xml version="1.0" ?>

<workflows>

    <workflow name="helloWorldWorkflow">

        <options>

            <enabled>true</enabled>

            <scheduler autorun="false" type="cron">

                <hours>*</hours>

                <minutes>*/0.1</minutes>

                <day>*</day>

                <month>11-12</month>

            </scheduler>

        </options>

        <steps>

            <step id="start">

                <id>start</id>

                <app>HelloWorld</app>

                <action>repeatBackToMe</action>

                <device>hwTest</device>

                <input>

                    <call format="str">Hello World</call>

                </input>

                <next step="1">

                    <flag action="regMatch">

                        <args>

                            <regex format="str">(.*)</regex>

                        </args>

                        <filters>

                            <filter action="length">

                                <args/>

                            </filter>

                        </filters>

                    </flag>

                </next>

                <error step="1"/>

            </step>

        </steps>

    </workflow>

</workflows>
Tag Description 
<workflows></workflows>
Container for Workflows 
<workflow></workflow>

Container for Workflow Object

Name = Name of the Workflow 

<options></options>
Container for Workflow Options 
<enabled></enabled>
Boolean value of whether the workflow should be run automatically when scheduled 
<scheduler></scheduler>

Container for scheduler object detailing when Workflow should be run automatically

Autorun=(true/false) - Enables/Disables scheduling options

type=(cron) - Determines scheduling type 

<hours></hours>
<minutes></minutes>
<day></day>
<month></month>

Containers for cron job actions

Wildcard can be used with *

Ranges can be specified with - (e.g. 11-12) 

<steps></steps>

Container for step objects 

<step></step>

Container for Step object

id=step ID

 
<id></id>

Step ID 

<app></app>

App Name 

<action></action>

Action name to be taken

 
<device></device>

Device Name (configuration profile) to be used 

<input></input>

Container for arguments

Tags contained within are specific to each app. 

<next></next>

Container for conditional

step=Step to jump to next if conditional is true 

<flag></flag>

Conditional statement object

action=function to verify output against 

<args></args>

Container for conditional arguments

Elements contained within are specific to function called

 
<filters></filters>

Container for filter objects

 
<filter></filter>

Container for filter object. Filters filter data before it is compared against a flag

Action=action to be called

 
<error></error>

Conditional statement object if error occurs

step=next step to jump to if true
