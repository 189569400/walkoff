A Brief Tour of the WALKOFF Interface

  • The purpose of this tutorial is to provide a brief overview of the different functions within WALKOFF and the interface. In-depth instructions and overviews for specific functionality can be found in subsequent documentation.

    The WALKOFF interface consists of the following pages:

    • Installed Apps - A drop down with all of the apps you have installed which lead to their custom interfaces
    • Controller - Displays which playbooks and workflows are loaded, allows you to create cases for logging and enables you to start, stop, pause, and resume the scheduler.
    • Playbook - Provides a drag and drop interface for you to create, design, and edit your custom workflows.
    • Devices - Allows you to manage your devices associated with the different apps. Think of devices as configuration profiles.
    • Triggers - Manages the WALKOFF triggers which enable workflows to be executed upon data being recieved.
    • Cases - An interface displaying the logs of configured cases.
    • Settings - Allows for the management of system configuration and user/role management.
