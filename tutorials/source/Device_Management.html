Devices Management

  • Each device has a few fields that are there by default. These fields are: device name, corresponding app, username, password, IP, and port. On top of these default fields, the user has the ability to dynamically add or change any additional fields.
  • The user can dynamically add these fields similar to how he would pass in the default fields, but with some minor differences.
  • The user needs to create a dictionary of extra fields, and when issuing a POST request to the add or edit device API call, the user should insert the extra fields as the string representation of the dictionary. Below is an example:
    • extraFields = {"extraKeyOne" : "extraValueOne", "extraKeyTwo" : "extraValueTwo"}
    data = {"name" : "testDeviceName", "extraFields" : json.dumps(extraFields)}
  • As you can see from above, the user must pass in the string representation extraFields dictionary via json.dumps().
  • In the back-end, these extra fields are stored as the string representation of the JSON dictionary. Once there is a call to edit any of these fields, or display a device, then that string representation is loaded as a JSON dictionary via json.loads(), edited or displayed as if the extra fields were there from the beginning, and then the JSON dictionary is stored again as a string via json.dumps().
