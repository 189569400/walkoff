Users and Roles Overview

  • Each user has a few fields associated with it. These fields are: a user's email (or username), password, and role (e.g. "admin" or "default"), which will be discussed in more detail below. Each user is stored in a SQLAlchemyUserDatastore, which includes various security measures, such as password encryption in the back-end database.
  • To add a user, you must make a "POST" request to "/users/add", including the user's username, password, and optional list of roles. There is also the ability to edit the password or role(s), remove the user, and display the user.
  • Each user can have one or more roles, which is essentially a set of permissions. For example, a user with an "admin" role may have the ability to remove a user, whereas a regular or "default" user may only have the ability to display a user.
  • When creating a new role, you will have the ability to provide a name and description for the role, as well as a list of permissions for that role. You may also leave the permissions blank to edit later, in which case only the default permissions will be loaded for that role. Once a role has been added, you can add that role to an existing user.
  • Roles are useful for providing an extra layer of security. You do not want to give anyone the ability to create a user with admin privileges, who would then be able to edit various configuration settings, workflow and trigger settings, and user settings.
