Flags

Flags are conditions which much be met before a step progresses to another. The condition which must be met is selected from a list functions defined in the core/flags directory. Each flag takes zero or more arguments and an input. When executing a workflow, the input to a flag is the output from the previous step. Additionally, data can be filtered through a Filter before being evaluated. Each flag can have its own list of filters, the output of the first being used as the input to the second and so on. The flags currently implemented are the following:

Count

Compares the number of elements in a collection or an integer to a value

Arguments
  • Threshold - The value to compare the input to
  • Operator - The method of comparison (<, <=, =, >=, or >)
Regular Expression Match

Attempts to compare the input to a regular expression

Arguments
  • Regular expression - The regular expression used to match the input against
